Configure global authorization signatures for notifications across all applications.

## Global Authorization

Set up universal notification authorization across all dApps:

```ts
// Global notification authorization
const setupGlobalNotificationAuth = async () => {
  const globalMessage = `Authorize notifications for all connected applications\n` +
                       `Wallet: ${modal.getAddress()}\n` +
                       `Timestamp: ${Date.now()}\n` +
                       `Scope: Global`
  
  try {
    const signature = await modal.signMessage(globalMessage)
    
    // Register global authorization with Reown Cloud
    const response = await fetch('https://api.reown.com/v1/auth/global', {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${projectId}`,
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        signature,
        message: globalMessage,
        address: modal.getAddress(),
        scope: 'global'
      })
    })
    
    const { globalToken } = await response.json()
    
    // Enable global notifications
    modal.enableGlobalNotifications(globalToken)
    
  } catch (error) {
    console.error('Global notification authorization failed:', error)
  }
}
```

## Cross-App Benefits

Global authorization provides:

- **Single Sign-On**: Authorize once, use everywhere
- **Consistent Experience**: Same notification preferences across dApps
- **Centralized Management**: Manage all notifications from one place
- **Reduced Friction**: No repeated authorization requests

## Privacy Controls

Users maintain control over global authorization:

```ts
// Revoke global authorization
const revokeGlobalAuth = async () => {
  await fetch('https://api.reown.com/v1/auth/revoke', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${globalToken}`
    }
  })
  
  modal.disableGlobalNotifications()
}
```

Global authorization streamlines the notification experience while preserving user privacy and control.

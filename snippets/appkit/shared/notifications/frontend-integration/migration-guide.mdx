Migration guide for upgrading to the latest AppKit Notifications API.

## Migration from v1 to v2

### Breaking Changes

1. **API Method Names**
   ```ts
   // Old (v1)
   modal.getNotificationHistory()
   modal.subscribeToNotifications()
   
   // New (v2)
   modal.getNotifications()
   modal.subscribeNotifications()
   ```

2. **Notification Object Structure**
   ```ts
   // Old (v1)
   interface OldNotification {
     id: string
     message: string
     timestamp: number
   }
   
   // New (v2)
   interface Notification {
     id: string
     type: 'transaction' | 'connection' | 'network' | 'custom'
     title: string
     message: string
     timestamp: number
     read: boolean
     data?: any
   }
   ```

### Migration Steps

1. **Update API Calls**
   ```ts
   // Replace old subscription method
   const unsubscribe = modal.subscribeNotifications((notifications) => {
     // Handle notifications
   })
   ```

2. **Update Notification Handling**
   ```ts
   // Update to handle new notification structure
   notifications.forEach(notification => {
     console.log(notification.title) // New field
     console.log(notification.type)  // New field
     console.log(notification.read)  // New field
   })
   ```

### New Features in v2

- Notification types and categories
- Read/unread status tracking
- Enhanced metadata support
- Improved persistence options
- Better error handling

Follow this guide to smoothly upgrade your notification implementation to the latest version.
